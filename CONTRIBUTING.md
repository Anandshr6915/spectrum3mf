# Contributing

Thanks for contributing to `spectrum3mf`.

## Ground Rules

- Keep pull requests focused and small.
- Open an issue before large architectural changes.
- Add or update tests for behavior changes.
- Do not include generated artifacts or large binary assets in PRs.

## Development Setup

```bash
python -m pip install -r requirements.txt
python -m pip install -e .
pytest
```

## Pull Request Checklist

- Clear title and summary of user-visible impact.
- Linked issue (if applicable).
- Tests added/updated and passing locally.
- No secrets, personal data, or unnecessary binaries.
